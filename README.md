# DeskPet · 把你的宠物变成陪你上班的桌面伙伴

> 一个**可换皮、可对话、可扩展**的桌宠 Agent。
> 丢一张你家宠物的（绿底精灵图）照片 → 它就变成桌面上会动、会陪你、会提醒你的小家伙。
> 支持「纪念模式」，给已经离开的毛孩子一个温柔的陪伴方式。

## 这是什么 / 差异化

市面上的开源桌宠大多在拼技术（记忆 / 情感 / RPG / 工具调用），但有两个共同盲区：

1. **换形象要美术重画** —— 它们是固定卡通形象，不是「你的」宠物；
2. **没人认真做情感和产品** —— 炫技但答不上「谁付费、为什么付费」。

DeskPet 反着来：
- **专属**：丢一张图就换皮（绿幕抠图 + 精灵图切帧），它是**你那只**宠物，不是通用形象；
- **情感**：纪念模式，定位「带着它形象的陪伴者」，不消费悲伤、不冒充本体；
- **可信**：AI 与密钥全在主进程；**工具调用走「规则意图路由 + 写操作确认 + trace 审计」**，AI 不会被幻觉误触发去动你的数据 —— 不乱来。

## 可信工具调用内核

桌宠不只是会聊天，它能帮你**记待办、看待办、报今日陪伴数据**，但每一步都受控（方法论沿用作者的 FitMind 项目）：

1. **意图由规则路由决定**，不交给模型 —— 工具不会被 AI 幻觉触发；
2. **读操作**（看数据/看待办）直接执行；**写操作**（记/改待办）必须你点「确认」才生效；
3. 模型只负责把结果**包装成自然语言**（配了 Key 时），没 Key 用模板兜底，离线也能用；
4. 每次调用写进**后台的 trace 审计日志**，谁、什么时候、动了什么，全程可追溯。

对桌宠说「记一下：买牛奶」「今天专注了多久」「有什么待办」试试。

## 安装使用

### 方式一：普通用户（一键安装）

到 [Releases](https://github.com/Andrew-JX/DeskPet/releases) 下载最新的 Windows 安装包：
- `DeskPet-Setup-x.x.x.exe` —— 安装版（带桌面快捷方式）
- `DeskPet-x.x.x.exe` —— 免安装便携版，双击即用

下载即用，无需任何环境。

### 方式二：开发者（从源码运行）

```bash
git clone https://github.com/Andrew-JX/DeskPet.git
cd DeskPet
npm install
npm start
```

### 自己打包安装包

```bash
npm run dist     # 在 dist/ 生成 Windows 安装包 + 便携版
```

> 维护者发版：打个 `v*` 标签（`git tag v0.1.0 && git push origin v0.1.0`），
> GitHub Actions 会自动构建并发布到 Releases。

首次启动会在右下角出现一只占位狗狗 🐶（无素材时的兜底）。
- **悬停** 宠物 → 出现互动按钮和 ⚙ 后台入口
- **单击** → 互动 + 随机气泡
- **双击** → 打开聊天输入框，和它对话
- **拖动** → 在桌面任意移动

- **示例素材**：仓库自带 [`samples/sample-dog-greenscreen.png`](samples/sample-dog-greenscreen.png)（程序生成的 5×4 绿底精灵图）。在桌面版后台「上传素材」选它，立刻看到换皮生效。
  - 想重新生成：`node tools/make-sample-sprite.js`

## 素材库 + 宠物与互动（核心玩法）

后台分两层：**素材库**（可复用素材）→ **宠物**（待机 + 自定义互动，素材都从库里挑）。

1. 点桌宠上的 **⚙** 打开后台
2. **素材库**：「上传素材到库」——整图、视频，或一张含多动作的**绿底精灵图**都行
   - 精灵图：设好「列/行」后点格子，把要的帧**切进库**（每帧成为一个可挑选素材）
3. **添加宠物** → 给它从库里选一个**待机素材**
4. **添加互动** → 改名（摸摸头 / 睡觉 / 摇尾巴…完全自定义）→ 从库里给它选素材
5. 桌面悬停宠物 → 出现你定义的互动按钮，点一下播对应素材，**几秒后自动回待机**
6. 可加**多个宠物**，桌面点「使用」切换

> 图片和视频就此统一：摸摸头用摇尾巴视频、睡觉用精灵图里趴着那一帧，各挑各的。
> 一张 20 动作的精灵图，切进库后可分别用作不同互动。

## 接入真实大模型（可选）

不配也能跑（自动退回随机回应）。要让它真的"会聊"：

```powershell
# Windows PowerShell
$env:ANTHROPIC_API_KEY = "你的key"
npm start
```

> **安全设计**：API Key 只读环境变量，**永不进入** config.json 或渲染层。
> 渲染层只能通过 `preload.js` 暴露的白名单 API 与主进程通信 —— 这是「可信桌宠」的基线。

## 目录结构

```
main.js        Electron 主进程：窗口、IPC、配置读写、AI 调用（密钥在此）
preload.js     contextBridge 白名单 API（渲染层唯一通道）
pet.html       宠物窗口：绿幕抠图 + 精灵图状态机 + 拖动 + 气泡 + 聊天
panel.html     后台：素材管理 / 网格映射 / 人设 / 番茄钟
assets/        config.json + user-media/（运行后生成）
```

## 状态机

精灵图按网格切帧，状态 → 帧区间映射（可在后台改）：

| 状态  | 触发                       | 默认帧 |
|-------|----------------------------|--------|
| idle  | 默认                       | 0–4    |
| walk  | 偶尔散步（移动窗口）       | 5–9    |
| happy | 点击 / 对话 / 提醒         | 10–14  |
| sleep | 闲置 30s                   | 15–19  |

## 已知降级 / 风险

- 视频/动图素材当前用占位兜底，精灵图是主路径
- 绿幕阈值偏宽容，复杂背景图需后台微调或换纯绿底
- `.mov` 能否播放取决于系统编码支持
