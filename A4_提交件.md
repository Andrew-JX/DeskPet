# DeskPet：把自己的宠物变成陪你上班的桌面伙伴

**方向：宠物 AI｜提交人：吉敏宇｜仓库/安装包：** https://github.com/Andrew-JX/DeskPet  
**一句话：**上传宠物绿底图/视频，生成一个可拖动、可互动、会提醒、可聊天、可纪念的桌面宠物。核心不是通用桌宠，而是“我自己的那只”。

## 1. 机会判断

**目标用户**：有宠物或曾养过宠物的电脑办公人群，尤其是独居、远程办公、长时间对屏幕工作，以及想纪念已离世宠物的人。  
**痛点**：现有桌宠多是固定卡通形象，和真实宠物无关；宠物照片停留在相册/壁纸里，不能进入日常工作流；离世宠物纪念多是静态摆件，缺少克制、可信、可长期打开的数字陪伴。  
**为什么现在做**：AI 生成、绿幕抠图、桌面 Agent 技术已足够低成本，能把“定制宠物形象”从昂贵美术活变成轻量生成链路；但现有桌宠多服务极客，普通养宠人与纪念场景仍空缺。

## 2. 用户证据

**平台观察**：抖音/小红书“桌面宠物、宠物桌宠、宠物纪念”相关内容有多条高互动样本；评论高频出现“怎么弄”“求教程”“能不能做我家猫/狗”。宠物纪念内容下，用户会主动讲述离世宠物故事并询问定制方式。信号不是“喜欢桌宠”，而是“想把自己的宠物放进去”。  
**竞品观察**：GitHub 高星桌宠/Agent 桌宠多强调记忆、RPG、工具调用，但换皮门槛高、安装偏极客；Clawd 等更面向程序员。机会缝隙在“普通用户一键用 + 专属宠物形象 + 情感边界”。  
**一手访谈**：访谈 10+ 位养宠/曾养宠朋友，普遍认可“自己的宠物形象”和“纪念模式”的情感价值；相比纯聊天，他们更想要“它在桌面动起来、提醒我休息、记得我的偏好”。

## 3. MVP 样例

已完成可运行 Electron Demo，并开放 GitHub 仓库与 Windows 安装包：

- **换皮核心**：上传绿底图片/循环视频，实时抠绿幕，显示为透明桌面宠物，可拖动、置顶、跨屏移动。
- **素材库与切帧**：上传精灵图后，拖框选区并按列/行切帧，复用为待机、点击、聊天、番茄钟、久坐提醒等动作。
- **互动与提醒**：点击宠物弹出互动菜单；支持自定义互动；番茄钟有“专注中”姿势和倒计时角标。
- **可信 AI 与记忆**：可填自己的模型 API Key（OpenAI 兼容接口或 Anthropic）开启真实多轮对话；支持最近对话记忆和长期记忆 notes；AI 可陪聊、记/查待办、查看专注数据；写操作必须确认，并写入 trace 审计；Key 只在主进程使用。
- **纪念模式**：定位为“带着它形象的陪伴者”，不冒充已故宠物本体，不编造主人没有说过的记忆。

## 4. 商业判断

**谁付费**：C 端为想要专属宠物桌宠/离世纪念的养宠上班族；B 端为宠物店、宠物医院、宠物殡葬/纪念服务商。  
**为什么付费**：免费桌宠很多，但“把我自己的宠物变成会动、会陪我工作的形象”有情感独占性；离世纪念场景付费意愿更强。  
**定价**：软件免费获客；专属形象生成/定制 19-39 元；纪念版（形象 + 寄语 + 动作包）99 元；B 端按套授权或按生成次数收费。  
**获客**：小红书/抖音发布“把我家宠物做成桌宠”“把离开的宠物留在桌面上陪我工作”等样片，引导到免费 Demo；再通过宠物社群、宠物店/医院合作获取种子用户。

## 5. 两周验证计划

**第 1 周：验证想不想要。**发布 3 条样片（普通陪伴 2 条、纪念向 1 条），同步投放抖音、小红书、宠物社群；访谈 20 位养宠用户，记录是否愿意上传宠物素材、想要哪些动作、是否接受本地记忆、能接受的价格。成功信号：单条 ≥50 收藏，或 ≥10 条“怎么做/能不能帮我做”的私信。  
**第 2 周：验证愿不愿付。**上线“上传照片/视频 -> 半自动生成专属桌宠 -> 交付安装包/素材包”的最小付费链路，跑 5-10 单真实订单。成功信号：≥5 单付费，或付费转化率 ≥10%。  
**资源投入**：1 人开发与内容即可启动；需要 AI 生成额度、少量内容投放预算、3-5 位种子用户素材授权；数据正向后补 1 名视觉/内容同学。

**MVP 取舍**：v1 不做任意本机软件调用、健康数据接入等高风险能力，先验证“专属换皮 + 陪伴提醒 + 可信 AI + 本地记忆 + 纪念边界”。
