# DeskPet · 把你的宠物变成陪你上班的桌面伙伴

**赛道：宠物 AI（轻量机会验证） | 提交人：吉敏宇 · AI 产品方向**
原型代码：本仓库 `npm install && npm start` ｜ 个人主页：https://mj-portfolio-gray.vercel.app/#/

---

### 1. 机会判断：目标用户 / 痛点 / 为什么现在值得做

**目标用户**：有宠物、或曾经养过宠物的上班族（每天 8h+ 对着电脑）。

**痛点**：① 独自办公缺陪伴，市面桌宠都是「通用卡通形象」，与我无关；② 宠物离世后的情感寄托，**几乎没有产品认真承接**。

**为什么现在做**：调研发现「桌宠 / Agent 桌宠」赛道已很卷（GitHub 上 Agentic-Desktop-Pet、Clawd on Desk 等多个高星项目），但它们有两个共同盲区 —— **形象不可换**（换皮要美术重画）、**只炫技不做情感与商业**（拼记忆/RPG/工具，答不上「谁付费」）。AI 立绘 + 绿幕抠图让「丢一张图就变成你那只宠物」第一次变得低成本可行，这是切入缝隙。

> **差异化一句话**：不拼 Agent 炫技，拼「**专属 + 情感 + 可信**」—— 它是**你那只**宠物，不是通用形象。

### 2. 用户证据

- **竞品即证据**：最高星的 Agentic-Desktop-Pet 自己在 README 标注 *"many bugs, please be patient"*，需 Python+Godot 自配环境，普通人装不起来；Clawd 只服务盯 Claude Code 的程序员 —— **全是给极客的玩具，无人切普通人 + 情感**。
- **需求热度**：小红书 / 抖音「宠物纪念」「把我家猫做成桌宠」类内容长期高互动，宠物离世后「想留个念想」是真实、高情绪强度的需求。
- **一手访谈（进行中）**：已向 10+ 位养宠/曾养宠的同事朋友（我的目标用户）求证，普遍认可「专属形象」和「纪念」两点的情感价值（详见 2 周计划，将补足样本与量化）。

### 3. MVP 样例（已可运行）

本仓库为可运行的 Electron 原型，`npm start` 即见右下角桌宠：

- **换皮**：后台上传绿底精灵图 → 自动绿幕抠图 + 按网格切帧 → 状态机播放（idle/walk/happy/sleep）；
- **陪伴**：透明无框置顶、可拖动；单击互动、双击对话（接 Claude，无 Key 自动退回本地文案）；
- **有用**：番茄钟 / 久坐提醒到点来叫你；
- **纪念模式**：语气更轻柔，定位「带着它形象的陪伴者」，不冒充本体、不消费悲伤；
- **后台管理**：素材 / 人设 / 提醒可视化配置，保存实时生效，预留扩展位。
- **可信基线**：AI 与 API Key 全在主进程，渲染层经 `preload` 白名单通信，密钥永不外泄。

### 4. 商业判断

| 维度 | 判断 |
|---|---|
| **谁付费** | 核心：想要「专属/纪念」形象的养宠上班族；延伸：宠物店/宠物医院（送客户的情感礼品）。|
| **为什么付费** | 免费桌宠到处有，但「**把我自己那只宠物变成会动的陪伴**」是情感独占价值，且离世纪念付费意愿最强。|
| **定价** | 软件免费获客；**按形象付费**：定制/生成一个专属精灵图 19–39 元，纪念版礼盒（含定制形象+寄语）99 元；B 端按套授权。|
| **获客** | 小红书/抖音内容自传播（「我把走丢的猫做成了桌宠」天然爆款逻辑）；宠物社群 + 宠物店异业合作。|

### 5. 两周验证计划

- **第 1 周 · 验证「想不想要」**：发布 3 条小红书/抖音样片（含纪念向 1 条），看收藏/私信转化；完成 20 份目标用户访谈，量化「愿为专属形象付费」比例与可接受价格。**指标**：单条 ≥50 收藏 或 ≥10 条「怎么做」私信即为正信号。
- **第 2 周 · 验证「愿不愿付」**：上线「照片→生成精灵图」最小付费链路（人工/半自动交付即可），跑 5–10 单真实付费；完善纪念模式语气与分寸。**指标**：≥5 单付费 = 方向成立，继续投入；偏离则调整定位或价格。
- **所需资源**：1 人即可启动（我）；AI 立绘生成额度 + 少量内容投放预算；若提速，需 1 名美术/内容协作。

---
*取舍说明：周期内**砍掉**「调用本机任意软件」等高风险能力，先把「专属换皮 + 陪伴 + 纪念」这一最小可信闭环做扎实 —— 先解决一个问题，再谈扩展。*
